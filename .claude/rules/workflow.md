# 用户指令处理流程

## 指令识别

用户说以下内容时触发：
- "我今天写了文章，内容在 {路径}"
- "我记录了 {内容}，在 {路径}"
- "有个笔记在 {路径}"

## 处理步骤

### 1. 读取内容
使用 Read 工具读取指定路径的文件

### 2. 判断类型

| 类型 | 存储位置 |
|------|----------|
| 可复用知识（技术文档、代码笔记、解决方案） | `knowledge/` |
| 临时操作记录 | `projects/` |
| 正式操作记录 | `active/` |

### 3. 存储

- **可复用知识** → `knowledge/{主题}/{文件名}.md`
- **临时记录** → `projects/YYYY-MM-DD.md`
- **正式记录** → `active/YYYY-MM-DD.md`

### 4. 更新操作记录（可选）

在 `active/YYYY-MM-DD.md` 中添加：

```markdown
## {操作简述}

- **类型**：写文章/整理笔记/添加知识
- **内容**：{标题/描述}
- **位置**：{路径}
- **来源**：{原始路径}
```
